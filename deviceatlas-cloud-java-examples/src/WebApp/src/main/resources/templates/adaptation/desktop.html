Content Adaptation

To adapt contents dynamically, so each visitor gets the optimum user experience. Whether the device used is a smartphone, tablet, eReader, or a low end device, you only get one chance to make a first impression. Ensure your site loads quickly and is easy to navigate. DeviceAtlas provides all the information needed to support content adaptation decisions.

On a page request, DeviceAtlas API is used to get the device properties. By putting conditions on some of the properties which are usually about the device type a layout template is chosen loaded wrapped around the data and sent back to the user.

Usually most of the contents remain the same and only the templates and styles are tuned for different devices.

Using the handful of DeviceAtlas properties each template can be fine-tuned for the device and browser

Image

Let's say we want to display a photo, as we now know the type of device viewing the page, we can select a photo which is the most suitable for the device. We can even optimize the page further and use property values such as "displayWidth", "displayHeight", "usableDisplayWidth" and "usableDisplayHeight" to provide the optimal photo.

As we already know a desktop computer is viewing the page we have plenty of room so we can show our largest photo.

Example directory and files

The web examples are based on a MVC model where files are placed inside /Examples/ExampleWebApp, having controllers in /src/main/java/com/deviceatlas/cloud/example/web and views in /src/main/resources.

All requests first go to the ControllerWebApp controller object. Here it is used DeviceAtlas to detect device type and decides which template should be used to wrap around the contents.
