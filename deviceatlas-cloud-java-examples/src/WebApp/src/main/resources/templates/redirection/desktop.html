Redirection Example

You have been redirected to this page because either you are using a desktop browser, a device that is masquerading itself as desktop to get a desktop experience or your machine is not detected as a desktop, tablet or mobile but you are being redirected to this website because this is the default one.

How it works

If we have different web sites and each one is created to give the best content and user experience to a specific device type. Users will enter our main web-site URL in their browsers, the source code which gets this first request will use DeviceAtlas CloudApi to detect the device type, then based on the conclusions, the user will be redirected to the web-site which provides the best experience for her/his device.

In this example four directories will mimic four different websites that give specific service to "desktop browsers", "mobile device" and "tablet pcs".

The web examples are based on a MVC model where files are placed inside /Examples/ExampleWebApp, having controllers in /src/main/java/com/deviceatlas/cloud/example/web and views in /src/main/resources.

All requests first go to the ControllerWebApp controller object. Here it is used DeviceAtlas to detect device type and decides which template should be used to wrap around the contents.

View of this page is templates/redirection/desktop.html
